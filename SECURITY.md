# 安全政策（Security Policy）

如果你发现了可能影响用户安全或数据安全的问题（例如 XSS、供应链风险、隐私泄漏等），请不要在公开 Issue 中披露细节。

## ✅ 推荐上报方式（优先）

- 使用 GitHub 的私密安全通告（Security Advisory）：
  - https://github.com/TUR1412/Guoman/security/advisories/new

## 🧾 报告内容建议

为了更快修复，请尽量包含：

- 影响范围与风险等级（你认为的严重性）
- 最小复现步骤（PoC）
- 修复建议（如有）
- 你期望的披露时间窗口（如有）

## ⛔ 不建议

- 不要在公开 Issue / PR 中贴出可直接被利用的 PoC（例如可复制执行的攻击脚本）
