# why · Diagnostics Imported Records Explorer

## 目标

在 **不破坏现有 Diagnostics 导入/导出能力** 的前提下，补齐“导入回放”的关键一环：

- 导入诊断包后，不仅能看摘要/对照指标，还能直接 **浏览导入包中的 logs/errors 记录**（筛选/检索/折叠/下载筛选结果）。
- 将导入入口收敛为单一、清晰的交互（拖拽导入 + 点击选择文件），避免多处入口与动作重复导致的 UI 噪音。

## 问题

当前 `/diagnostics` 已支持导入 `.json` / `.json.gz` 诊断包并展示摘要对照，但存在可用性缺口：

- 只能看到导入包的 **统计摘要**，无法直接查看导入包里具体的 logs / errors 内容，导致“对照排障”的价值被削弱（需要复制 JSON 自行搜索）。
- 导入入口与清理动作存在重复区域（多个入口/按钮），降低了页面的“仪表盘”聚焦感。

## 成功标准（可验证）

1. 导入诊断包后，页面出现“导入日志浏览器/导入错误浏览器”，支持：
   - 关键词检索（message/source/context 或 message/source/stack）
   - 日志级别筛选（ERROR/WARN/INFO/DEBUG）
   - 折叠展开详情（stack/context）
   - 下载筛选结果（JSON）
2. 导入入口统一：仅保留在“导入诊断包”区域内的 DropZone（点击/拖拽均可），移除其它重复入口。
3. 全程 local-first：不新增任何远程上报/上传/外链依赖。
