# 变更提案: 量子未来体验与内容智能扩展

## 需求背景

现有国漫世界已具备完整的本地化体验与视觉系统，但在“未来感审美、内容智能深度、观影规划能力”三方面仍有明显提升空间。当前需要在不破坏核心架构与逻辑的前提下，引入更强的内容理解与个人化呈现，并完成一次高标准的视觉与交互跃迁。

## 产品分析

### 目标用户与场景

- **用户群体:** 国漫深度爱好者、轻度浏览用户、设计审美敏感型用户
- **使用场景:** 追更计划、推荐探索、长内容挑选、站内沉浸浏览
- **核心痛点:** 缺乏观影规划、推荐解释不足、标签/工作室洞察弱、视觉风格尚未达到顶级审美

### 价值主张与成功指标

- **价值主张:** 用“内容智能 + 未来视觉 + 观影规划”形成可持续留存的体验闭环
- **成功指标:** 推荐点击率提升、追更完成度提升、页面停留时长提升、Lighthouse 得分接近满分

### 人文关怀

所有推荐/画像完全本地计算；视觉增强提供弱网/低动效/可调节通道，避免过度刺激与视觉疲劳。

## 变更内容

1. 未来感视觉体系升级（字体、色板、背景纹理、卡片质感、按钮层级）
2. 智能观影计划器（追更时间预算、剩余集数/时长、完成预计）
3. Taste DNA 2.0（匹配度评分 + 推荐理由展示）
4. 标签趋势热力（热门标签 + 质量/热度维度）
5. Studio Radar（工作室维度的高口碑雷达）
6. Audience Pulse（作品口碑与评论热度摘要）
7. 性能与代码清洗（加载时序、渲染分支、样式复用）

## 影响范围

- **模块:** Design System / Recommendations / Following / Insights / Anime Detail / UI Components
- **文件:** src/assets/styles/global.css, src/pages/_, src/components/_, src/utils/\*, README.md
- **API:** 无新增后端 API（本地计算）
- **数据:** 新增本地持久化键（观影计划参数）

## 核心场景

### 需求: 智能观影计划器

**模块:** Following / Watch Planner
在追更中心输入每日观看预算后，系统计算剩余集数与预计完成日期。

#### 场景: 计划生成

用户已有追更内容与观看进度。

- 展示剩余时长、完成天数与优先队列

### 需求: Taste DNA 2.0

**模块:** Recommendations / AnimeCard
用户进入推荐页时看到匹配度与推荐理由。

#### 场景: 匹配解释

用户可直观看到“喜欢的标签/制作方”触发的推荐原因。

- 推荐卡片显示匹配度与关键词

### 需求: 标签趋势热力

**模块:** Recommendations
用户希望从标签维度洞察整体趋势。

#### 场景: 热力探索

系统展示热门标签与质量/热度指标，支持点击跳转。

- 标签热力卡片清晰表达趋势

### 需求: Studio Radar

**模块:** Insights
用户希望了解高口碑制作方分布。

#### 场景: 工作室洞察

系统列出高评分/高人气工作室与代表作。

- 可引导进入详情页继续探索

### 需求: Audience Pulse

**模块:** Anime Detail
用户进入作品详情时希望快速理解口碑。

#### 场景: 口碑摘要

系统展示评论密度、热度指数与关键词摘要。

- 帮助判断是否值得观看

### 需求: 未来视觉与性能

**模块:** Design System
用户在首页与主导航体验未来感视觉体系。

#### 场景: 首屏沉浸

强化背景层次与交互层级，确保低动效与弱网可控。

- 不破坏可读性与性能指标

## 风险评估

- **风险:** 视觉与交互改动范围大，存在可读性/性能风险
- **缓解:** 使用可配置 token、保持数据层与路由不变、覆盖弱网与 reduced motion 分支
