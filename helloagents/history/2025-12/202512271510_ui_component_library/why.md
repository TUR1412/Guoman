# Why - UI Component Library（国漫视觉组件库）

## 目标

- 将项目中高频出现的 UI 容器/布局/反馈模式抽象为可复用原语（Grid / Stack / Card / Dialog / Skeleton / Toast）。
- 建立一套可运行的 Storybook，用“组件级演示”替代散落在页面里的样式试错，提高迭代速度与一致性。
- 为后续“视觉回归”打基础：让 UI 可被脚本化地渲染与验证，避免改一处风格牵连全站但无法快速确认。

## 现状痛点

- 组件层存在大量“相似但不完全一致”的布局与容器写法（max-width、gap、grid、卡片基座等），一旦 tokens/交互策略变化，需要多点修改。
- 动效与容器的组合需要统一入口：尤其在 `forceReducedMotion` 等全局策略开启时，组件应当自动收敛到一致行为。
- 缺少组件级可视化工作台，导致 UI 变更主要依赖“运行整站 + 手动点页面验证”，成本高、遗漏概率大。

## 成功标准

- 新增 `src/ui/`：提供至少 6 个可复用原语，并在站内关键组件中落地使用（不只停留在 demo）。
- 新增 `.storybook/`：可在本地通过 `npm run storybook` 启动并展示核心原语 Stories。
- 质量闸门 `npm run check` 通过（不把 Storybook 引入带来的噪音带进主构建链路）。
